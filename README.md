# Teams Meeting Insights MCP Server

An MCP server that syncs Microsoft Teams meeting data (transcripts, AI insights, action items) into Elasticsearch so Claude can search your meeting history.

Runs as two Docker containers — Elasticsearch and a Node.js server — with a web dashboard for Microsoft sign-in.

## Quick Install

```bash
curl -fsSL https://raw.githubusercontent.com/WayneJetski/teams-meetings-mcp/main/scripts/install.sh | bash
```

This clones the repo, prompts for your Azure credentials, starts the containers, and adds `teams-mcp` to your PATH.

Options:

```bash
# Install to a specific directory
curl -fsSL ... | bash -s -- --dir ~/tools/teams-mcp

# Use HTTPS instead of SSH for git clone
curl -fsSL ... | bash -s -- --https
```

Once installed, manage the server with:

```
teams-mcp              # Start the server (default)
teams-mcp stop         # Stop containers
teams-mcp logs         # Tail container logs
teams-mcp update       # Pull latest + rebuild
teams-mcp status       # Container & health status
teams-mcp uninstall    # Remove teams-mcp completely
teams-mcp help         # Show all commands
```

## Manual Setup

### Prerequisites

- Docker and Docker Compose
- Node.js 20+
- An Azure AD app registration (see [Azure Setup](#azure-ad-setup) below)

### 1. Clone and start

```bash
git clone https://github.com/WayneJetski/teams-meetings-mcp.git
cd teams-meetings-mcp
./scripts/start.sh
```

On first run, the script will create `.env` from the template and ask you to fill in your Azure credentials:

```bash
AZURE_TENANT_ID=your-tenant-id
AZURE_CLIENT_ID=your-client-id
AZURE_CLIENT_SECRET=your-client-secret
```

Once those are set, run `./scripts/start.sh` again. It will build and start the Docker containers and register the MCP server with Claude Code.

### 2. Sign in

Open http://localhost:4005 and sign in with your Microsoft account. Your first sync runs automatically after sign-in.

### 3. Use with Claude

The start script auto-configures the MCP server in Claude Code. Restart Claude Code to pick it up. Claude will then have access to these tools:

| Tool | Description |
|---|---|
| `search_meetings` | Full-text search across summaries, transcripts, action items, decisions |
| `get_meeting` | Full details of a specific meeting by ID |
| `get_action_items` | Action items, filterable by owner and date range |
| `get_decisions` | Decisions, filterable by keyword and date range |
| `meeting_stats` | Aggregated stats: meeting count, top attendees, topics |

## Azure AD Setup

### 1. Register an app

1. Go to **Azure Portal** > **Entra ID** > **App registrations** > **New registration**
2. Name: `Teams Meeting Insights MCP` (or whatever you like)
3. Supported account types: **Single tenant**
4. Redirect URI: **Web** — `http://localhost:4005/auth/callback`

### 2. Add API permissions

Go to **API permissions** > **Add a permission** > **Microsoft Graph**:

**Delegated permissions:**

| Permission | Purpose |
|---|---|
| `OnlineMeetings.Read` | List the signed-in user's meetings |
| `OnlineMeetingTranscript.Read.All` | Read meeting transcripts |
| `Calendars.Read` | Discover meetings via calendar |
| `User.Read` | Read signed-in user profile |
| `OnlineMeetingAiInsight.Read.All` | AI insights (requires Copilot license) |

Click **Grant admin consent** for your org.

### 3. Create a client secret

Go to **Certificates & secrets** > **New client secret** > copy the value into your `.env`.

## Data Tiers

Set `GRAPH_DATA_TIER` in `.env` to control what gets synced:

| Value | License Required | What it pulls |
|---|---|---|
| `transcripts` (default) | M365 E3/E5/Business | Raw `.vtt` transcripts |
| `insights` | M365 Copilot | Structured AI insights (notes, action items, decisions) |
| `both` | M365 Copilot | Both — AI insights as primary, transcripts for deep search |

## Environment Variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `AZURE_TENANT_ID` | Yes | — | Azure AD tenant ID |
| `AZURE_CLIENT_ID` | Yes | — | Azure AD app client ID |
| `AZURE_CLIENT_SECRET` | Yes | — | Azure AD client secret |
| `SESSION_SECRET` | Auto | — | Auto-generated by start script |
| `GRAPH_DATA_TIER` | No | `transcripts` | `transcripts`, `insights`, or `both` |
| `SYNC_CRON` | No | `0 * * * *` | Cron schedule for auto-sync (default: hourly) |
| `SYNC_LOOKBACK_DAYS` | No | `30` | How far back to look on first sync |

## Project Structure

```
src/
├── index.js                  # Express + MCP + cron startup
├── config.js                 # Environment variable loading
├── elasticsearch.js          # ES client, index mappings, queries
├── public/index.html         # Web dashboard (sign-in UI)
├── auth/
│   ├── routes.js             # OAuth routes (/auth/login, /callback, /logout)
│   └── requireAuth.js        # Session-based route protection
├── graph/
│   ├── auth.js               # MSAL confidential client (token management)
│   ├── client.js             # Graph API HTTP wrapper
│   ├── meetings.js           # Meeting discovery via calendar
│   ├── transcripts.js        # Transcript fetch + VTT parsing
│   └── insights.js           # AI insights (Copilot tier)
├── sync/
│   ├── engine.js             # Core sync orchestration
│   └── scheduler.js          # Cron-based scheduling
├── mcp/
│   ├── server.js             # MCP server (streamable HTTP)
│   └── tools/                # One file per MCP tool
├── api/
│   └── router.js             # REST endpoints (sync, ingest, health)
└── utils/
    ├── vttParser.js          # VTT → structured JSON
    └── timestamps.js         # Date helpers
```

## Teardown

```bash
docker compose down          # stop containers, keep data
docker compose down -v       # stop containers, delete all data
```
